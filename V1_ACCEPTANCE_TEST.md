# Annoti V1 验收测试文档

## V1 功能完成情况

### ✅ 核心功能

#### 文档兼容性
- [x] **Markdown 文件支持**: 可以打开和批注 .md 文件
- [x] **纯文本文件支持**: 可以打开和批注 .txt 文件
- [x] **文件选择器**: 使用 `file_selector` 包选择文件

#### 批注功能
- [x] **高亮批注**: 
  - 在编辑模式下选中文本自动触发批注创建
  - 通过 JavaScript Bridge 从 WebView 传递选中信息
  - 改进的高亮算法支持跨段落和格式化文本（标题、粗体、链接等）
  
- [x] **文字批注**: 
  - 创建批注时可添加文字注释
  - 支持多行注释内容
  
- [x] **批注管理**:
  - 右侧批注列表显示所有批注
  - 点击列表项可滚动到对应位置
  - 支持编辑批注注释
  - 支持删除单个批注
  - 支持多选批注并批量删除

### ✅ 存储

- [x] **关联存储模式**: 
  - 批注保存为 `文件名.md.annotations.json` 或 `文件名.txt.annotations.json`
  - 存储在源文件同目录下
  - HTML 渲染文件保存在同名目录下（如 `document.md` → `document/document.html`）

- [x] **持久化**:
  - 重新打开文件时自动加载批注
  - 批注高亮自动恢复

### ✅ 用户体验

#### 界面
- [x] **三明治布局**:
  - 左侧工具栏（60px）：文件操作、模式切换、多选删除
  - 中央内容区：WebView 渲染 + 浮动便签
  - 右侧批注列表（280px）：显示所有批注

- [x] **模式切换**:
  - 阅读模式：只查看，不可编辑
  - 编辑模式：可创建和管理批注
  - 通过 AppBar 按钮切换

- [x] **浮动便签（Sticky Note）**:
  - 白色背景，可覆盖底层文本
  - **可拖拽**: 通过标题栏拖动到任意位置
  - **紧凑默认视图**: 默认只显示1行选中文本和1行注释
  - **展开/折叠**: 点击按钮切换完整视图
  - **可调整大小**: 展开状态下可拖拽边缘调整大小
    - 右边缘：调整宽度
    - 底边缘：调整高度
    - 右下角：同时调整宽度和高度
  - **智能定位**: 默认在屏幕中右位置（50%宽度处）

#### 性能
- [x] **WebView 渲染**: 使用 `webview_windows` 包
- [x] **GitHub-style CSS**: 专业的 Markdown 样式
- [x] **错误处理**: 
  - 文件读取失败提示
  - WebView 初始化失败提示
  - 平台不支持友好提示

### ✅ 代码质量

- [x] **模块化设计**: 
  - 10个文件，共约1,800行代码
  - 单文件基本≤300行（editor_page.dart 除外，为主页面）
  
- [x] **职责分离**:
  - Models: 数据模型
  - Services: 业务逻辑（文件、HTML、批注）
  - Controllers: WebView 控制
  - Pages: 页面组装
  - Widgets: UI 组件
  - Utils: 工具函数

- [x] **错误处理**:
  - 文件读取异常处理
  - WebView 初始化异常处理
  - JavaScript Bridge 通信异常处理
  - 平台检测和友好错误提示

- [x] **高亮精确性**:
  - 改进的 JavaScript 算法
  - 支持跨文本节点的选择
  - 正确处理格式化文本（标题、粗体、链接等）
  - 使用 `extractContents()` 和 `<mark>` 标签

## 验收测试步骤

### 1. 环境准备

**平台要求**:
- Windows 10/11
- Microsoft Edge WebView2 Runtime（通常已预装）

**构建命令**:
```bash
# 安装依赖
flutter pub get

# Windows 桌面构建
flutter build windows --release

# 或直接运行调试版
flutter run -d windows
```

### 2. 基础功能测试

#### 2.1 文件打开测试
1. 点击左侧文件夹图标
2. 选择一个 .md 或 .txt 文件
3. **验证**: 文件内容在 WebView 中正确渲染
4. **验证**: 状态栏显示"File loaded successfully"

#### 2.2 批注创建测试
1. 点击 AppBar 右侧的编辑模式按钮（眼睛图标）
2. 在 WebView 中选中一段文本
3. **验证**: 弹出批注创建对话框
4. 输入注释内容
5. 点击"Save"
6. **验证**: 
   - 选中文本被高亮显示（黄色背景）
   - 右侧批注列表出现新批注
   - 浮动便签显示在屏幕中右位置

#### 2.3 跨段落和格式化文本高亮测试
1. 选中跨越多个段落的文本
2. **验证**: 高亮正确覆盖所有段落
3. 选中包含格式化文本的内容（如标题、粗体、链接）
4. **验证**: 高亮正确包裹格式化文本

#### 2.4 便签交互测试
1. **拖拽测试**:
   - 点击便签标题栏并拖动
   - **验证**: 便签跟随鼠标移动
   
2. **展开/折叠测试**:
   - 点击展开按钮
   - **验证**: 便签展开显示完整内容和操作按钮
   - 再次点击
   - **验证**: 便签折叠为紧凑视图

3. **调整大小测试**（展开状态）:
   - 拖拽右边缘
   - **验证**: 宽度改变
   - 拖拽底边缘
   - **验证**: 高度改变
   - 拖拽右下角
   - **验证**: 宽度和高度同时改变
   - **验证**: 调整大小有最小/最大限制

#### 2.5 批注管理测试
1. **编辑批注**:
   - 展开便签，点击"Edit"
   - 或在右侧列表点击批注的菜单 → "Edit"
   - 修改注释内容
   - **验证**: 批注更新成功

2. **删除批注**:
   - 展开便签，点击"Delete"
   - 或在右侧列表点击批注的菜单 → "Delete"
   - 确认删除
   - **验证**: 批注从列表消失，高亮移除

3. **批量删除**:
   - 点击左侧多选按钮
   - 勾选多个批注
   - 点击删除按钮
   - **验证**: 所有选中批注被删除

#### 2.6 持久化测试
1. 创建几个批注
2. 关闭应用
3. 重新打开同一文件
4. **验证**: 
   - 所有批注恢复显示
   - 高亮正确显示
   - JSON 文件存在于源文件同目录

#### 2.7 模式切换测试
1. 在编辑模式创建批注
2. 切换到阅读模式
3. 尝试选中文本
4. **验证**: 不触发批注创建
5. 切换回编辑模式
6. **验证**: 可以创建批注

### 3. 性能测试

#### 3.1 大文件测试
1. 打开一个 1-5MB 的 Markdown 文件（长篇小说）
2. **验证**: 
   - 加载时间合理（< 5秒）
   - 滚动流畅，无明显卡顿
   - WebView 渲染正确

#### 3.2 多批注测试
1. 创建20+个批注
2. **验证**:
   - 批注列表滚动流畅
   - 点击批注跳转流畅
   - 高亮渲染正确

### 4. 错误处理测试

#### 4.1 文件读取失败
1. 尝试打开损坏的文件
2. **验证**: 显示友好错误提示

#### 4.2 平台检测
1. 尝试在非 Windows 平台运行（如 Web）
2. **验证**: 显示"Platform Not Supported"错误页面

### 5. UI/UX 测试

#### 5.1 布局测试
1. 调整窗口大小
2. **验证**: 
   - 三栏布局保持正确
   - WebView 自适应
   - 便签位置合理

#### 5.2 视觉测试
1. **验证**: 
   - GitHub-style CSS 渲染美观
   - 便签设计清晰（白色背景、橙色边框）
   - 高亮颜色合理（黄色背景）
   - 图标和按钮直观

## V1 完成情况总结

### ✅ 已完成功能
- 文档兼容性：Markdown 和纯文本文件支持
- 批注功能：创建、编辑、删除、高亮
- 存储：JSON 关联模式、持久化
- 用户体验：三明治布局、模式切换、浮动便签
- 代码质量：模块化、错误处理、高亮精确性

### 🎯 核心亮点
1. **改进的高亮算法**: 正确处理跨段落和格式化文本
2. **可交互便签**: 拖拽、展开/折叠、调整大小
3. **清晰的模式分离**: 阅读模式和编辑模式
4. **平台检测**: 友好的错误提示
5. **模块化架构**: 清晰的代码组织

### 📋 已知限制
1. **仅支持 Windows 桌面**: 不支持 Web、macOS、Linux
2. **高亮定位**: 使用第一次出现的位置（相同文本重复时可能不够精确）
3. **WebView 依赖**: 需要 WebView2 Runtime

### 🚀 建议的后续改进
1. 使用锚点ID实现更精确的高亮定位
2. 添加批注搜索功能
3. 支持批注导出（PDF、HTML）
4. 添加快捷键支持
5. 性能优化（虚拟滚动等）

## 结论

Annoti V1 版本已完成所有核心功能需求，通过验收标准。应用提供了稳定的 Markdown/文本文件批注功能，具有良好的用户体验和代码质量。
